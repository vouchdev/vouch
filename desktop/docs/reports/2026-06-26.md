## Summary

Stood up `vouch-desktop` from nothing to a working, verified v0.1.0 in one day:
a cross-platform Electron app that puts a GUI on the **entire vouch `kb.*`
command surface (all 54 methods)** plus the dual-solve runner. vouch is driven as
an unmodified child process over its existing JSONL and review-ui transports, and
the review gate is preserved — the UI never auto-approves. The app launches,
auto-opens a KB, and renders all 11 views with zero runtime errors against the
real `vouch` 0.1.0 binary.

## Completed today

**Discovery & design**
- Mapped vouch's full surface via a parallel agent workflow: all 54 `kb.*`
  methods with parameters/types, the JSONL wire protocol, the launch/auth/env
  contract, and the HTTP + WebSocket review-ui surface.
- Produced the architecture design doc (`docs/architecture.md`) incl. a
  method-by-method coverage table.

**Backend / process spine (main process)**
- `jsonl-client` — spawns `vouch serve --transport jsonl`, NDJSON framing,
  id-correlated request/response, per-method timeouts (long ops get 10 min).
- `supervisor` — health-polls `kb.status`, restarts the child with backoff.
- `vouch-locator` — resolves the binary (configured path → bundled-frozen →
  PATH → `python -m vouch`), git probe, `vouch init`.
- `http-client` — lazily spawns `vouch review-ui --allow-dual-solve` only for the
  dual-solve view; drives `/dual-solve/run|job|choose` + `/ws` progress.
- `kb-store`, `ipc`, `tray` + `Notifier`, `index` (lifecycle, single-instance).

**Renderer (buildless, sandboxed)**
- Data-driven form generator off a verified parameter catalog — typed controls,
  enum dropdowns (sourced from vouch's own model enums), sliders, tag inputs,
  JSON editor, native file pickers, search-backed id typeaheads.
- Result renderers (claim/page/entity/relation/source/proposal cards, search
  hits, context pack, synthesis, audit timeline, lint/doctor findings, graph
  code, JSON-tree fallback).
- 10 method-driven views + a bespoke Dual-Solve view (run → live progress →
  side-by-side diffs → choose), a Dashboard, a Review queue with approve/reject,
  and a detail drawer. **Capability-gating** against the connected vouch.

**Packaging & hygiene**
- `electron-builder.yml` (dmg / nsis / AppImage), `gen-methods.js` generator,
  catalog-coverage test suite, JSONL smoke harness, README, CHANGELOG, LICENSE.

## Key decisions

- **Separate repo, vouch untouched** — honors vouch's buildless/no-npm rule by
  keeping the Node toolchain entirely out of the Python package.
- **JSONL for everything, HTTP only for dual-solve** — the review queue runs over
  JSONL, so the app is fully functional without vouch's `[web]` extra.
- **Buildless renderer** (no Vite/TS) — runnable and verifiable immediately; Vite
  remains an easy later upgrade.

---

## Blockers / risks

- **Capability gap (handled):** installed vouch 0.1.0 advertises 43/54 methods;
  the other 11 exist only in newer source. The UI gates against the live
  capabilities list, so this degrades gracefully and self-heals on upgrade.
- **Packaging unproven:** bundling a frozen vouch (PyInstaller) for zero-Python
  installs is scaffolded but not built/tested cross-OS — flagged as the top risk
  for the packaging milestone.
- **Env quirk (local only):** this sandbox sets `ELECTRON_RUN_AS_NODE=1`; must be
  stripped to launch. Not an issue on a normal user machine.
