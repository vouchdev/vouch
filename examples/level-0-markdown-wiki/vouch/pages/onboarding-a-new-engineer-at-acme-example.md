This page is the running notes for getting a new engineer productive in
their first week. It is a plain wiki page: no review-gated claims, no
graph edges, just prose someone typed up.

## Day one

- Get a laptop from it-example and set up SSO.
- Clone the `acme-example/platform` monorepo.
- Read the architecture overview page and the local-dev page.

## First week

- Pair with your onboarding buddy on a small starter ticket.
- Ship one change to staging by Friday. It does not have to be big.
- Add yourself to the team roster page.

## Who to ask

- Build/CI questions: ask in `#eng-build`.
- Access/permissions: file a ticket with it-example.
