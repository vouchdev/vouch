Vouch

The memory your agent has to earn.

A local-first knowledge base for coding agents with one rule: every write passes review. The agent proposes, you approve, and the claim lands in your repo — cited, diffable, and remembered next session.

Quick Start

Full guide →
vouch — quick start
$ pipx install vouch-kb
$ cd your-repo && vouch init
$ vouch serve # mounts the MCP server

Works anywhere Python 3.10+ does. The command installed is vouch; the distribution on PyPI is vouch-kb. Mining Gittensor? Seed the SN74 decision-memory with vouch init --template gittensor.

What It Does

How the gate works →

Review Gate

Nothing becomes memory until a different identity signs it. Self-approval is refused by the model, not by convention.

Plain Files on Disk

Claims are YAML, pages are markdown, all under .vouch/ in your repo. They diff, grep, and review like code.

Committed Audit Trail

Every propose, approve, reject, and supersede appends to an audit log that travels with the repository.

Three Surfaces, One Gate

MCP for agents, a CLI for humans, JSONL for pipes — every surface calls the same reviewed lifecycle.

Search That Cites

SQLite FTS5 with optional embeddings. Every result carries the source it was approved with.

Recall on Session Start

kb_context opens each session with the decisions that survived review — no re-deriving March's fix in July.

Straight From the Audit Log

every write is an event
proposal.approved ✓

“emission split favors merged OSS work over the issue treasury”

signed alice-example · cites docs/gittensor.md
proposal.rejected ✗

“scoring-weight claim arrived without a source”

refused: no citation registered — provenance is schema, not policy
claim.superseded ↺

“retry budget raised from 2 to 3”

old claim kept, marked superseded — the diff stays on disk

Sample entries. Every decision — including the refusals — is appended to a committed, append-only audit.log.jsonl.
